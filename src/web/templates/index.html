Routing engine (D5 cross-border opt-in)

Default us uses the US-only OSRM (Tier 1 baseline). Pick us_canada for trips where Canadian highways are genuinely faster (Detroit↔Buffalo, Niagara↔Sault Ste M). The cross-border engine penalizes each detected border-crossing leg by 2 × border_crossing_minutes of customs time.

Time-budgeted mode (Tier 2 headline · score-maximization)

Setting total_trip_days switches the solver from state coverage to score maximization within a soft time budget. Each POI has a value computed from poi_priority (per-POI override) ↓ category_priority (category fallback) ↓ 0. Read priority as "hours of driving I'd accept to visit this POI."

Higher overage penalty = harder cap.

category_priority (table)

Set a priority for each category you care about. Categories with no value are treated as 0 (incidental — visited only if on the way).
